Styling

Styling is used to change the formatting of an HTML element. CSS syntax consists of the following:

  • Selector
  • Property
  • Value
  • Declaration

Colors

Color property can be set with one of the following options:

  • HEX: {color: #006400;}
  • RGB: {color: rgb(230, 45, 23);}
  • HSL: {color: hsl(120, 60%, 70%);}

See this link for a list of common color examples. Note, hexacedimals where pair numbers are the same can be shortened to 3 digit number (e.g. Aqua can be coded as #00FFFF or #0FF)

HSL stands for Hue-saturation-lightness. Value of hue is between 0 and 360, whereas saturation and lightness are percentages (e.g. color: hsl(120, 60%, 70%);). HSL is conveneint for adjusting colors.

To change opacity, use HSLA or RGBA color scheme where "a" is the opacity value from 0 to 1. For zero opacity, can use "transparent" keyword for the color property (i.e. color: transparent;).

Fonts

Fonts fall into 2 categories: Serif and San-Serif (with and without markings respectively). Fallback fonts are used when specified font may not be available on the computer. The following syntax is required:


{font-family: ""Garamond", "Times", serif;}


The fallback syntax indicates that Garamond should be used first followed by Times if Garamond is not available and by any serif font if niether Times nor Garamond is available. Note that font names must be in double-quotes but serif does not need to be.

It is also possible to link font that is likely not installed on the computer to Google Fonts directory at fonts.google.com. Link to selected font is embedded in the header of the HTML document using href, family, type and relattributes. Multiple families can be included using family=Droid+Serif|Playfair+Display designation.

Another way to link a font is through Font-Face CSS property. Put google link into browser, copy each "latin" font and paste into CSS file at the top of the stylesheet. You can also modify @font-face property to use local font files as follows: @font-face { font-family: "Glegoo"; src: url("../fonts/Glegoo-Regular.ttf") format('truetype'); }

The Box Model

The Box Model is a set of properties that define part of element that take up space on the page. The Box Model consists of the following properties:

  • Width & height specifies width + height of the content area; measured in px or % (e.g. 600px or 100%)
  • Padding specifies space between content and border; apply using padding property or padding-top, padding-right, padding-left, padding-bottom if want unequal sides (e.g. 10px) OR padding: 13px 11px 15px 6px property starting with top and ending with left OR shortcut padding: 10px 5px if top/bottom and right/left are equal.
  • Border specifies thickness and style of border around the content; border is set with specific width, style and color (e.g.{border: 3px solid coral;}). Can also set border shape with border-radius (e.g. 5px or 100%) property.
  • Margin specifies amount of space between the border and outside edge of the element (i.e. how much space will be between one element and other elements). Same syntax as padding is used for margins including abbreviations. To center content on a page margin: 0 auto; declaration can be used in combination with width property. NOTE: top/bottom or vertial margins between elements collapse so that the margin of the bigger element become the entire margin between the 2 elements.

max-width, min-width, max-height, min-height are used to set minimum and maximum sizes for elements when a webpage is expanded or contracted. Overflow property is used to prevent text overflow if height/width of the element are too small for the content. The property is set at parent level and the values are as follows:

  • hidden: overflow content is not visible
  • scroll: scrollbar is added to the content
  • visible: default value; content is visible outside the element

Many browsers have default values for margin and padding which limits styling. Use * {margin: 0; padding: 0;} code to reset these defaults.

Box-sizing property controls default box size model browser will use. The values are as follows:

  • content-box: default value; affected by border thickness and padding
  • border-box: resets box model on HTML page to a type that does not have sizing issues. Use *{box-sizing: border-box;} code at the beginning of the CSS styling page.

Positioning

Position property specifies position of an element in the following three ways:

  • relative: position is relative to the element's defualt position with top/bottom/right/left properties following it (value in px) and measured in em, px or %
  • static: default position of the element
  • absolute: positions element within and relative to its ancestor element (e.g. div)
  • fixed: element will not move on a page (good for headers). top/bottom/right/left property can be used to position fixed element as well

See header and side bar for examples of fixed, absolute and relative position property

Ancestor element: relative position
Child element: absolute position

Z-index property controls how far back or forward overlapping elements with relative positionining should appear using integer values (e.g. -2, -1, 2). An element with greater stack order is always in front of an element with a lower stack order. If two positioned elements overlap without a z-index specified, the element positioned last in the HTML code will be shown on top.

Centering Elements

The following methods can be used to center elements on a page:

Relative Measurements

Relative measurements allow for responsive design of a page so it looks good when browser window is made bigger or smaller. Relative measurements include the following

  • em is an alternative unit of measurement to px and represents size of base font (e.g. base font is 12px so 2em = 24px). It is most useful for sizing elements compared to parent element.
  • rem is an alternative unit of measurement to px and represents size of base font of root element (i.e. html tag). It is most useful for sizing elements consistently across the entire website. To use rem, need to set font size for html element (i.e. html {font-fize: 16px;}).
  • percentages are used to set width/height, padding, border and margins of an element relative to its parent element. Dimensions of the parent element need to be set first. Dimensions of the child element should not be 100% or it may overflow. Note: vertical padding and margin are calculated based on the width of the parent element.
  • min-width and max-width ensure a minimum and maximum width for an element when a browser page is resized to allow for ease of reading. Measurement is in pixels.
  • min-width and max-width ensure a minimum and maximum height for an element. Content might overflow outside the element.
.container {
width: 50%;
height: 200px;
overflow: hidden;
}

.container img {
max-width: 100%;
height: auto;
display: block;
}

Media Queries

Media queries are used to change website design when screen is resized using media features. Media features detect many aspects of a browser including:

  • screen width and height
  • resoultion
  • orientation and more

Operators and and or are used to indicate when all features or only one of the features needs to be used respectively. Media queries should be set at natural website breakpoints.

  • @media only screen and (min-width: 320px) and (max-width: 480px) {
    body {
    font-size: 12px;
    }
    }
  • @media only sreen and (max-width: 480px), (orientation: landscape) {
    h1 {
    font-size: 4rem;
    }
    }
